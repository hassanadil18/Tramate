# Discord Channel Setup Guide for Admins

## Overview
This guide explains how admins can add and configure Discord channels in the Tramate platform.

## 3 Required Fields for Discord Integration

When setting up a Discord channel, you need exactly **3 required pieces of information**:

### 1. Discord Server ID (Guild ID)
- **How to get it:** Right-click on the Discord server name → Select "Copy Server ID"
- **Example:** `123456789012345678`
- **What it's for:** Identifies which Discord server the channel belongs to

### 2. Discord Channel ID
- **How to get it:** Right-click on the specific channel → Select "Copy Channel ID"
- **Example:** `987654321098765432`
- **What it's for:** Identifies the exact channel where trading signals are posted

### 3. Channel Access Role ID
- **How to get it:** Right-click on the role → Select "Copy Role ID"
- **Example:** `456789012345678901`
- **What it's for:** The role users must have to access this specific channel

## Step-by-Step Setup

### Prerequisites
- Enable Developer Mode in Discord:
  1. User Settings → Advanced
  2. Enable "Developer Mode"
  3. Now you can right-click and copy IDs

### 1. Access Admin Dashboard
- Log in as admin user
- Navigate to `/admin`
- Click on "Channels" in the sidebar

### 2. Create New Channel
- Click "New Channel" button
- Fill in the form:

#### Basic Information:
- **Name**: Channel display name (e.g., "Premium Signals")
- **Channel Type**: Select "Discord"
- **Status**: Set to "Active"
- **Monthly Price**: Set subscription price
- **Description**: Brief description of the channel

#### Discord Configuration (Required Fields):
1. **Discord Server ID**: Paste the Guild ID from step 1 above
2. **Discord Channel ID**: Paste the Channel ID from step 2 above
3. **Channel Access Role ID**: Paste the Role ID from step 3 above

#### Optional Settings:
- **Discord Invite Link**: Permanent invite URL for users
- **Webhook URL**: Auto-generated by the system

### 3. Save and Auto-Setup
- Click "Save" to create the channel
- The system will automatically:
  - Create a Discord webhook
  - Test the connection
  - Verify bot permissions

### 4. Verify Setup
- Check the channel details page
- Ensure all 3 required fields show valid IDs
- Click "Test Connection" to verify bot access
- Should show "Discord connection successful!"

## What Happens After Setup

### Automatic Features:
- **Real-time Message Monitoring**: Bot watches for trading signals (BUY, SELL, LONG, SHORT)
- **Member Verification**: Checks if users have the required role
- **Access Control**: Grants/revokes access based on Discord membership
- **Signal Processing**: Automatically processes and executes trades

### Admin Dashboard Shows:
- ✅ All 3 required fields with visual indicators
- ✅ Webhook status and connection health
- ✅ Subscriber count and activity
- ✅ Recent signals and trades

## Troubleshooting

### Common Issues:

1. **"Required field missing"**
   - Ensure all 3 fields (Server ID, Channel ID, Role ID) are filled
   - Check that IDs are copied correctly (no extra spaces)

2. **"Discord connection failed"**
   - Verify bot is added to the Discord server
   - Check if channel ID is correct
   - Ensure bot has required permissions

3. **"Failed to create webhook"**
   - Bot needs "Manage Webhooks" permission
   - Check if channel allows webhooks
   - Verify bot role hierarchy

4. **"User verification failed"**
   - User doesn't have the required role
   - Role ID might be incorrect
   - User may have left the server

### Getting Help:
- Check Admin → Logs for detailed error messages
- Use "Test Connection" button for diagnostics
- Verify all 3 required fields are correctly set

## Security Notes
